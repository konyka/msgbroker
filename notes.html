msgbroker Development Notes

High-performance messaging library in pure C (C99-C23) — reimplementation of nanomsg SP protocols with memory pool, thread pool, coroutine, io_uring, and distributed capabilities.

Current Status: Phases 0-23 complete. 40/40 tests passing (Debug + Release). 20 socket types, 3 transports, io_uring, distributed cluster, 4 benchmarks, scatter/gather I/O, cmsg ancillary data, coroutine I/O, timeout/non-blocking support, version API, cross-platform CI (Linux/macOS/Windows). Clean build with -Wall -Wextra.

Nanomsg Analysis Summary

The original nanomsg has 211 source files across 19 directories. Key findings:

  • 6 protocols, each with user-level + raw/X-level implementations (20 socket types total)
  • 4 transports: inproc, IPC, TCP, WebSocket
  • AIO subsystem: FSM-based async, single worker thread (pool is TODO), epoll/kqueue/poll backends
  • Message: nn_chunkref (inline small / heap large), nn_msg (sphdr + hdrs + body)
  • VTable-based OOP: sockbase_vfptr, pipebase_vfptr, transport vtable
  • Max 512 sockets hardcoded, single global lock for socket creation

Supplementary: Raw/X-Variant Protocols

Implemented 10 raw (x-variant) protocol socket types for device forwarding. These have no state machines, no NOSEND/NORECV flags, and no EFSM guards — they allow direct pipe-level send/recv for building message forwarding devices.

  • [x] XPAIR — bidirectional, no state (like PAIR but raw)
  • [x] XPUSH — lb send, can also recv (raw PUSH)
  • [x] XPULL — fq recv, can also send (raw PULL)
  • [x] XREQ — lb send, no state machine (raw REQ)
  • [x] XREP — fq recv + lb, no state machine (raw REP)
  • [x] XPUB — broadcast send + recv (can receive subscriptions)
  • [x] XSUB — fq recv + send (can send subscriptions)
  • [x] XBUS — broadcast to all + recv from any (raw BUS)
  • [x] XSURVEYOR — broadcast + recv, no survey state (raw SURVEYOR)
  • [x] XRESPONDENT — recv + send, no last_pipe tracking (raw RESPONDENT)
  • [x] MB_SOCKTYPE_FLAG_RAW (4) added to protocol.h
  • [x] All 20 socktypes (10 user + 10 raw) registered in global.c
  • [x] Socket type IDs: user variants at +0/+1, raw variants at +8/+9 within each protocol's 16-space

Phase 20: Performance Optimization

Optimized the inproc hot path for maximum throughput:

  • [x] mb_msg_init_data() — New function combining init+memcpy in a single call. mb_send() now uses it instead of separate mb_msg_init() + memcpy(), avoiding a redundant memset of the body chunkref.
  • [x] Lazy eventfd signaling — mb_efd now tracks a watched flag set only when a caller retrieves the FD via mb_efd_getfd() (for MB_SNDFD/MB_RCVFD poll integration). mb_efd_signal() skips the write() syscall when no one is watching. This eliminated 4 syscalls per round-trip in the benchmark (send sndfd + rcvfd, recv sndfd + rcvfd).
